MCP (Model Context Protocol) என்றால் என்ன?

சுருக்கமாகச் சொன்னால், MCP என்பது AI மாடல்களையும் (உதாரணமாக: Claude, ChatGPT போன்றவை) உங்கள் கம்பெனியின் தரவுகளையும் (Data sources / Tools) பாதுகாப்பாகவும் எளிதாகவும் இணைப்பதற்கான ஒரு "பொதுவான மொழி" அல்லது "பாலம்" (Open Standard / Universal Protocol) ஆகும். இதை Anthropic நிறுவனம் உருவாக்கியுள்ளது.


எளிய உதாரணம் (Layman Example): யுனிவர்சல் பிளக் அடாப்டர் (Universal Plug Adapter)

நீங்கள் வெளிநாடுகளுக்குப் பயணம் செய்வதாக கற்பனை செய்துகொள்ளுங்கள்.

    முன்பு நிலைமை (Without MCP): ஒவ்வொரு நாட்டிலும் (அமெரிக்கா, இங்கிலாந்து, ஆஸ்திரேலியா) வெவ்வேறு வகையான பிளக் பாயிண்டுகள் இருக்கும். எனவே, உங்கள் லேப்டாப்பை சார்ஜ் செய்ய ஒவ்வொரு நாட்டுக்கும் தனித்தனி அடாப்டர்களை வாங்க வேண்டும்.
    AI உலகிலும் இதுதான் நடந்தது. ஒரு AI-ஐ GitHub உடன் இணைக்க ஒரு தனி கோட் (Custom API integration), Google Drive-உடன் இணைக்க வேறொரு கோட், Slack-உடன் இணைக்க இன்னொரு கோட் என டெவலப்பர்கள் ஒவ்வொரு டூலுக்கும் தனித்தனியாகக் கஷ்டப்பட்டு இணைப்புப் பாலங்களை உருவாக்க வேண்டி இருந்தது.

    இப்போது (With MCP): MCP என்பது ஒரு "யுனிவர்சல் டிராவல் அடாப்டர்" (Universal Adapter) போன்றது. இதை ஒருமுறை உங்களிடம் வைத்துவிட்டால் போதும், எந்த நாட்டுப் பிளக் பாயிண்டாக இருந்தாலும் எளிதாக இணைத்துவிடலாம்.
    அதேபோல, MCP என்ற இந்த ஒற்றைப் பாலத்தைப் பயன்படுத்தி, AI-ஐ எந்தவொரு டேட்டாபேஸ், பைல் சிஸ்டம் அல்லது டூலோடு (GitHub, Jenkins, Slack, Local Files) எந்தப் புதிய கஸ்டம் கோடும் எழுதாமலேயே மிக எளிதாக இணைத்துவிட முடியும்.


MCP எப்படி வேலை செய்கிறது? (மூன்று முக்கியப் பகுதிகள்)

இதன் கட்டமைப்பை மூன்று எளிய பகுதிகளாகப் பிரிக்கலாம்:

    MCP Host (பயனர் பயன்படுத்தும் AI): இது நீங்கள் கேள்விகளைக் கேட்கும் AI அப்ளிகேஷன் (உதாரணமாக: Claude Desktop அல்லது நீங்கள் உருவாக்கும் AI ஏஜென்ட்).

    MCP Server (பாலம்): இதுதான் ஒவ்வொரு தரவுத்தளத்துடனும் (Data source) நேரடியாகப் பேசும் சிறிய புரோகிராம். (உதாரணமாக: Google Drive-க்கு ஒரு MCP Server, GitHub-க்கு ஒரு MCP Server).

    Data Sources (தரவுகள்): AI-க்குத் தேவையான உண்மையான தகவல்கள் இருக்கும் இடம் (உங்கள் கம்ப்யூட்டரில் உள்ள பைல்கள், கம்பெனி டேட்டாபேஸ் போன்றவை).

நீங்கள் AI-யிடம் ஒரு கேள்வி கேட்கும்போது, அது MCP Server-ஐத் தொடர்புகொண்டு, பாதுகாப்பான முறையில் அந்தத் தகவலை எடுத்துவந்து உங்களுக்குப் பதிலளிக்கும்.
ஏன் MCP மிக முக்கியமானது?

    பாதுகாப்பு (Security): AI உங்கள் எல்லா டேட்டாவையும் காப்பி செய்து வைத்துக்கொள்ளாது. உங்களுக்கு எப்போது, எந்தத் தகவல் தேவையோ, அதை மட்டும் MCP மூலம் பாதுகாப்பாகப் படித்துக்கொள்ளும்.

    எளிமையான வேலை (Easy Integration): டெவலப்பர்கள் ஒவ்வொரு முறை புதிய AI வரும்போதும் புதிதாக இன்டக்ரேஷன் (Integration) செய்யத் தேவையில்லை. ஒருமுறை MCP Server-ஐ உருவாக்கிவிட்டால், எந்த AI மாடலாக இருந்தாலும் அதைப் பயன்படுத்திக் கொள்ள முடியும்.
